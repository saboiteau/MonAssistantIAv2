## Pense-betes
- **Paradoxe estimation IA** : IA facilite estimation projets (Operum/Idealink génèrent plans détaillés en minutes) MAIS estimer projets IA reste complexe (20k$ chatbot basique → 500k$+ systèmes avancés). Coûts cachés : data acquisition/cleaning, talents (100-200k$/an ingénieur IA), infrastructure GPU. Gartner : 40% projets IA annulés d'ici 2027 (coûts excessifs, ROI incertain).
- **AWS Kiro - Specification-Driven Development (2025)** :
  - Antidote chaos vibe coding, 3 phases : Requirement Unpacking → Technical Design Generation → Task Implementation
  - Synchronisation continue specs ↔ code (docs jamais obsolètes)
  - Pricing : Free (50 interactions/mois), Pro (19$/mois, 1k interactions), Pro+ (39$/mois, 3k interactions)
- **GitHub Spec-Kit (open-source)** :
  - "Retourne script dev traditionnel" : specs exécutables (vs consultatives)
  - CLI universelle (Claude Code, Copilot, Cursor, Gemini, Windsurf)
  - 5 commandes : /constitution, /specify, /plan, /tasks, /implement
  - Pont vibe coding chaotique ↔ dev structuré enterprise
- **Three Fidelities Framework (Kieran Klaassen, Cora)** :
  - Insight : "AI made us sloppy because it made us forget how to plan"
  - Fidelity 1 (5%) : Quick fix (1-line, typos, bugs évidents) → planning minimal
  - Fidelity 2 (80%) : Sweet spot (multi-fichiers, refactoring) → **ROI massif planification** (20min recherche = heures debugging évitées)
  - Fidelity 3 (15%) : Big uncertain (features majeures, scope flou) → vibe planning (3+ prototypes rapides)
  - Exemple Email Bankruptcy : 53k emails, 3 prototypes difficulté croissante → apprentissage → décomposition Fidelity 2
- **"Plans teach systems, code solves problems"** : MANTRA CENTRAL
  - Plans = savoir institutionnel réutilisable/transférable
  - Après 50+ reviews, IA reflète préférences architecturales équipe automatiquement
  - Code résout problème ponctuel, plans construisent intelligence organisationnelle durable
- **CC-Workflows Brainstorming Skill (Claude)** :
  - 4 phases : Autonomous Recon → Understanding → Exploration (2-3 approches + trade-offs) → Design Documentation
  - Évite engagement prématuré à architecture sans exploration alternatives
- **NIST AI RMF** : Evolution conformité facultative → standard central gouvernance IA (détails non développés dans chunks lus)
- **Outils mentionnés** : GitHub Spec-Kit, CC-Workflows Brainstorming, AWS Kiro, SuperClaude, Operum, Idealink, Wardley Mapping IA
- **Recommandations implémentation** :
  1. Gouvernance d'abord (NIST AI RMF, processus validation, métriques)

  3. Bons outils (Spec-Kit, Kiro, SuperClaude, Operum/Idealink)
  4. Mesurer/itérer (ratio planning/coding, réduction bugs, ROI)

## RésuméDe400mots

WEnvision présente un guide complet 2025 sur **AI4Project**, transformation de la gestion projet par l'IA. Article dense compilant 7+ frameworks/outils avec thème central : **la planification devient avantage concurrentiel à l'ère IA**.

**Paradoxe estimation** : IA facilite estimation projets traditionnels (Operum/Idealink génèrent plans/budgets en minutes) MAIS estimer projets *IA* reste notoirement complexe (20k$ chatbot basique → 500k$+ systèmes avancés). Coûts cachés explosent : data acquisition/cleaning, talents rares (100-200k$/an ingénieur IA), infrastructure GPU. **Gartner prédit 40% projets IA annulés d'ici 2027** pour coûts excessifs/ROI incertain. Solution : approche méthodique transformant gestion de réactive → proactive, planification = activité plus fort levier.

**AWS Kiro (2025)** : Specification-Driven Development réconciliant vitesse vibe coding + rigueur enterprise. Workflow 3 phases : Requirement Unpacking → Technical Design Generation → Task Implementation. Synchronisation continue specs ↔ code prévient documentation obsolète. Pricing accessible : Free (50 interactions/mois), Pro (19$/mois, 1k), Pro+ (39$/mois, 3k). Efficace projets complexes nécessitant traçabilité complète/conformité.

**GitHub Spec-Kit** : Outil open-source "retournant script dev traditionnel" en rendant specs exécutables (vs consultatives). CLI universelle compatible agents IA majeurs (Claude Code, Copilot, Cursor, Gemini, Windsurf). 5 commandes : /constitution (gouvernance), /specify (requirements), /plan (stratégies), /tasks (décomposition), /implement (développement). Pont parfait chaos vibe coding ↔ dev structuré enterprise. Supporte natif contraintes organisationnelles, conformité, modernisation legacy.

**Three Fidelities Framework** (Kieran Klaassen, GM Cora) : Insight "AI made us sloppy because it made us forget how to plan." Catégorise travail pour router effort approprié :
- **Fidelity 1 (5%)** : Quick fix (1-line, typos) → planning minimal, exécution directe
- **Fidelity 2 (80%)** : Sweet spot (multi-fichiers, refactoring) → **ROI massif planification** (20min recherche = heures debugging évitées)
- **Fidelity 3 (15%)** : Big uncertain (features majeures, scope flou) → vibe planning (3+ prototypes rapides)

Exemple **Email Bankruptcy Cora** : 53k emails, 3 prototypes difficulté croissante → apprentissage → décomposition en pièces Fidelity 2.

**Mantra central** : "**Plans teach systems, code solves problems.**" Plans accumulent savoir institutionnel réutilisable/transférable. Après 50+ reviews, IA reflète automatiquement préférences architecturales équipe. Code résout problème ponctuel, plans construisent intelligence organisationnelle durable.

**CC-Workflows Brainstorming Skill** : Claude skill transformant concepts flous → designs complets. 4 phases : Autonomous Recon (analyse artifacts avant questions) → Understanding (collecte purpose/contraintes) → Exploration (2-3 approches architecturales + trade-offs) → Design Documentation. Évite engagement prématuré architecture sans exploration alternatives.

**Recommandations implémentation** : (1) Commencer gouvernance (NIST AI RMF, validation, métriques), (2) Planification systématique (CC-Workflows, Three Fidelities, formation, documentation plans), (3) Bons outils (Spec-Kit, Kiro, SuperClaude, Operum/Idealink), (4) Mesurer/itérer (ratio planning/coding, bugs post-planning, ROI).

Conclusion WEnvision : **Planification = nouveau différenciateur stratégique**. Organisations maîtrisant planification systématique IA construisent avantage concurrentiel durable vs celles adoptant IA sans rigueur.
